## Terms

- `project_base`
  - Stage-3 formal project object produced from public structured parsing.
- `rule_hit`
  - Stage-4 formal rule result backed by boundary note and evidence references.
- `project_fact`
  - Stage-6 unified formal fact object consumed by page, API, sales, contact, and delivery surfaces.
- `sale_gate_status`
  - Formal sellability state from stage 6: `OPEN / REVIEW / HOLD / BLOCK`.
- `review_status`
  - Formal review state: `PENDING / CONFIRMED / REJECTED / OVERRIDDEN`.
- `report_status`
  - Formal report state: `DRAFT / READY / ISSUED / REVOKED`.
- `authority-critical chain`
  - A chain that must be protected by contracts and integration tests because it carries formal business truth.
- `current task`
  - The single live execution task recorded in `docs/governance/CURRENT_TASK.yaml`.
