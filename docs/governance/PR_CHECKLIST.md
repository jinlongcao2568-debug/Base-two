# AX9S PR_CHECKLIST

每个 PR 在提交和合并前必须逐项确认。

## 1. 范围检查
- [ ] 本次 PR 只有一个主目标。
- [ ] 改动目录都在任务授权范围内。
- [ ] 没有顺手修改无关模块。
- [ ] 没有新增未审批顶级目录。

## 2. 结构检查
- [ ] 没有新增第二套解析真相层。
- [ ] 没有新增第二套主判断或事实面。
- [ ] 页面层没有写业务规则。
- [ ] 规则层没有写抓取逻辑。

## 3. 字段与接口检查
- [ ] 新字段已登记到 `SCHEMA_REGISTRY.md`。
- [ ] 接口变更已同步 `INTERFACE_CATALOG.yaml`。
- [ ] breaking change 已明确标注。

## 4. 数据库检查
- [ ] 如涉及表结构变更，已新增 migration。
- [ ] 已提供 rollback 说明。
- [ ] 已提供 seed/demo fixture。

## 5. 测试检查
- [ ] 单元测试通过。
- [ ] 集成测试通过。
- [ ] 改阶段 3 已补 fixture。
- [ ] 改阶段 4 已补规则命中测试。
- [ ] 改阶段 5/6 已补快照测试。

## 6. 文档检查
- [ ] 权威文档不需要同步或已同步。
- [ ] 治理文件不需要同步或已同步。
- [ ] 模块 README 已更新。

## 7. 风险检查
- [ ] 已说明风险。
- [ ] 已说明未完成项。
- [ ] 已说明回滚方式。
