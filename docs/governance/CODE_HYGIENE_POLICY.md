# AX9S Code Hygiene Policy

## 目标

- 让自动化开发在可维护范围内运行
- 让文件体量、函数体量和跨阶段混写具备可执行门禁
- 让 `python scripts/check_hygiene.py` 成为开发前后固定检查项

## 代码行数门禁

### `src/**/*.py`

- 告警阈值：`>300` 行
- 阻断阈值：`>500` 行

### `tests/**/*.py`

- 告警阈值：`>500` 行
- 阻断阈值：`>700` 行

## 函数体量门禁

- 单函数逻辑行数告警阈值：`>50`
- 单函数逻辑行数阻断阈值：`>80`

说明：

- 逻辑行数忽略空行、纯注释行和仅包含文档字符串边界的行
- `check_hygiene.py` 以 AST 的 `lineno/end_lineno` 和源码文本统计逻辑行

## 主职责约束

- 一个文件只承载一个主职责
- 第一版自动检查使用两类启发式：
  - 文件体量与公开符号体量过大时给出告警
  - 在某阶段目录中实现另一个阶段的核心职责逻辑时直接阻断

允许项：

- 通过约定承接点进行消费和连接
- 在测试与集成验证中引用上下游阶段对象

禁止项：

- 在 `src/stageX_*` 文件中实现 `stageY_*` 的主裁决逻辑
- 在非承接点文件里混入跨阶段核心职责

## 自动化执行要求

- 开发前：
  - `python scripts/check_repo.py`
  - `python scripts/check_hygiene.py`
- 开发后：
  - `python scripts/check_hygiene.py`
- 关账前：
  - 阻断项必须清零
  - 告警项允许保留，但必须在 runlog 中给出说明

执行口径：

- 开发前：只阻断明显超限和跨阶段核心职责混入
- 开发后：告警和阻断都要记录
- 关账前：只允许带说明保留告警，不允许保留阻断项

## 当前限制

- 第一版不会自动重构超长文件
- 第一版不会自动判断复杂设计层面的职责拆分，只提供硬阈值和明显越界检测

## 测试侧执行口径

- automation/governance 场景测试优先提取 builder 或 helper，避免把 registry、worktree 和 review bundle setup 直接堆进单个测试函数。
- 不允许通过放宽 `check_hygiene.py` 阈值来消除测试 warning。
- 当前热点文件采用 `warning 清零 + 防回归测试` 策略；后续若重新出现同类 warning，优先抽取场景 builder，而不是接受长期噪音。
