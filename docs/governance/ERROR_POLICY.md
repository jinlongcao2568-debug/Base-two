# AX9S ERROR_POLICY

## 1. 目标
本文件定义抓取、解析、规则、报告与事实面中的错误处理边界，避免静默失败和错误升级。

## 2. 错误等级
### E1：可恢复错误
示例：单页抓取失败、临时网络错误、截图失败。
处理：允许重试；记录日志；不得静默吞掉。

### E2：可降级错误
示例：某公开环节缺失、部分字段无法解析、同名无法排除。
处理：允许降级为 `CLUE` 或 `OBSERVATION`；必须写边界说明。

### E3：不可降级错误
示例：结构化对象无法生成、阶段 6 事实面写入失败、迁移失败。
处理：任务失败；必须阻断继续输出。

## 3. 禁止行为
- 不得静默吞异常。
- 不得把 E3 包装成正常输出。
- 不得把内部权限缺失写成公开硬命中。
- 不得在页面层掩盖系统级失败。

## 4. 规则结果降级原则
- 公开证据不足以支撑正式命中：降级为 `CLUE`。
- 信息非常不充分，只能观察：降级为 `OBSERVATION`。
- 无法公开回链：不得产出 `AUTO_HIT`。

## 5. 人工复核触发条件
以下情况必须进入人工复核：
- 多源信息冲突
- 同名未排除
- 证据等级只能到 C
- 导出前包含 B 级线索
- 报告结论可能被误读为终局法律认定
