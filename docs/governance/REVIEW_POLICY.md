# AX9S REVIEW_POLICY

## 1. 目的
规定哪些变更必须人工 review，不能只靠 AI 或自动化门禁放行。

## 2. 必须人工 review 的改动
以下任何一类出现，必须人工 review：
1. 目录结构变化
2. 新增或删除接口
3. breaking change
4. 新增、重命名或删除正式字段
5. 表结构迁移
6. 删除旧兼容字段
7. 规则结论口径变化
8. 报告模板变化
9. 阶段 6 事实面聚合逻辑变化
10. 公开边界规则变化

## 3. 建议人工 review 的改动
- 高价值专题逻辑变化
- 大于 500 行有效代码改动
- 跨两个以上相邻阶段改动
- 规则结果与历史样本明显不一致

## 4. review 最低结论要求
review 结果必须明确：
- 是否通过
- 风险点
- 是否需要补测试
- 是否需要补文档
- 是否需要拆分任务后重提 PR
